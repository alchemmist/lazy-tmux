⟫ Lazy session restore for tmux. fzf‑popup, zero‑overhead, JSON on disk.

Install GitHub Docs

why lazy-tmux?

lazy loading

Sessions stay on disk as JSON. Only the one you pick via fzf is restored. No autorestore of 50 unused sessions.

fzf‑popup

prefix+f → fuzzy find over all saved sessions. Select, press enter, it’s back. Instant.

auto‑save

Background daemon saves every 5 minutes. Manual prefix+s also works. Plain JSON, no surprises.

tmux start

Last attached session is automatically restored when tmux starts. Zero config after hook.

XDG compliant

All data in ~/.local/share/lazy-tmux/. No dotfile clutter in your project dirs.

static binary

Written in Go, depends only on fzf. No python, no npm – just a single binary.

⚡ quick install

bash
# latest linux binary
curl -LO https://github.com/youruser/lazy-tmux/releases/latest/download/lazy-tmux-linux-amd64
chmod +x lazy-tmux-linux-amd64
sudo mv lazy-tmux-linux-amd64 /usr/local/bin/lazy-tmux

# or via go
go install github.com/youruser/lazy-tmux/cmd/lazy-tmux@latest
tmux.conf
# lazy-tmux — minimal integration set -g @lazy-tmux-autorestore 'on' set-hook -g client-attached 'run-shell "lazy-tmux restore-last"' bind s run-shell "lazy-tmux save" bind f run-shell "lazy-tmux select" # reload config (prefix + r is recommended) # tmux source-file ~/.tmux.conf

⎋ commands & usage

prefix + ssave all current tmux sessions (immediate)
prefix + fopen fzf popup with all saved sessions → restore on enter
lazy-tmux restore <name>restore specific session from disk
lazy-tmux restore-lastrestore most recently used session
lazy-tmux daemonstart auto‑save daemon (5m interval)
lazy-tmux listlist all saved session names
